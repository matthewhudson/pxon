The Idea

What is PXON?

PXON (pronounced "picks on") is a JSON-based format for describing pixel art. Think of it as the love child of EXIF metadata and your favorite 8-bit canvas. It gives your pixel art a structured, portable, machine-readable home — complete with artist info, creation metadata, and every single pixel coordinate.

The format was created by Jenn Schiffer and uses two main objects: exif for image metadata (borrowed from the Exif RDF schema), and pxif — the pixel art spin on EXIF — which holds the actual pixel data.

{}

Just JSON

No custom binary format. No special parser. It's JSON all the way down. Paste it, POST it, store it, diff it.

px

Pixel Perfect

Every pixel is an object with x, y, and color. The smallest unit of art, described precisely.

i

Rich Metadata

Track artist, software, copyright, description, and timestamps. Your pixel art deserves provenance.

Fun fact: PXON was born from the same creative spirit that gave us make8bitart.com — a beloved pixel art painting app used by thousands of artists and bored-in-meetings professionals alike.

exif Properties

Metadata about the artwork, inspired by the Exif RDF Schema:

Property
Type
Description
software
string
The tool used to create/export the PXON
artist
string
The identity of the pixel artist
imageDescription
string
A description of the pixel artwork
userComment
string
A freeform comment from the creator
copyright
string
Copyright info for the artwork
dateTime
string
When the PXON was initially generated

pxif Properties

The pixel art data itself. Contains a pixels array where each pixel is:

Property
Type
Description
x
integer
The x coordinate position of the pixel
y
integer
The y coordinate position of the pixel
color
string
The color of the pixel (e.g. rgba(0, 0, 0, 1))

Note: The size property was removed from the canonical spec (v0.2.0) since a pixel is, by definition, the smallest unit. The PXON library still supports it as a non-canonical convenience field, defaulting to 1.

The Library

Getting Started

The pxon npm package gives you a PXON class to create, import, manipulate, and export pixel art data. It's tree-shakable, ships ESM + CJS bundles, includes full TypeScript types, and has zero runtime dependencies.

Install

# Pick your package manager
pnpm add pxon
npm install pxon
yarn add pxon

Quick Start

import { PXON } from 'pxon'

// Create a new PXON instance
const art = new PXON()

// Set metadata about your artwork
art.artist = 'Your Name'
art.software = 'https://make8bitart.com/'
art.imageDescription = 'A tiny pixel heart'
art.dateTime = new Date().toISOString()

// Draw some pixels!
art.setPixel({ x: 1, y: 0, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 2, y: 0, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 0, y: 1, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 1, y: 1, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 2, y: 1, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 3, y: 0, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 4, y: 0, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 3, y: 1, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 4, y: 1, color: 'rgba(255, 0, 80, 1)' })
art.setPixel({ x: 5, y: 1, color: 'rgba(255, 0, 80, 1)' })

// Export as a PXON-formatted JSON object
const data = art.export()
console.log(JSON.stringify(data, null, 2))

Import Existing PXON

import { PXON } from 'pxon'

const art = new PXON()

// Import from a PXON JSON object (e.g. fetched from an API)
art.import({
  exif: {
    artist: 'jenn schiffer',
    software: 'https://make8bitart.com/',
    imageDescription: 'tiny face',
  },
  pxif: {
    pixels: [
      { x: 0, y: 0, color: 'rgba(0, 0, 0, 1)', size: 1 },
      { x: 2, y: 0, color: 'rgba(0, 0, 0, 1)', size: 1 },
      { x: 1, y: 2, color: 'rgba(255, 0, 0, 1)', size: 1 },
    ]
  }
})

// Now you can query, modify, and re-export
const pixel = art.getPixel(0, 0)   // { x: 0, y: 0, color: 'rgba(0, 0, 0, 1)', size: 1 }
const all = art.getAllPixels()     // Array of all pixel objects
console.log(art.artist)            // 'jenn schiffer'

Reference

API Reference

The PXON class is the only export. Click each method to expand its details.

constructor new PXON() ▶

Creates a new PXON instance with empty metadata and no pixels.

import { PXON } from 'pxon'
const art = new PXON()
.import() import(data: PXONImportData): void ▶

Imports PXON-formatted data into the instance. Accepts an object with optional exif, pxif, and dataURL fields. Overwrites any existing data.

art.import({
  exif: { artist: 'you', software: 'my-app' },
  pxif: { pixels: [{ x: 0, y: 0, color: 'rgba(0,0,0,1)', size: 1 }] }
})
.export() export(): PXONExportData ▶

Returns a plain object containing all EXIF metadata, pixel data, and the optional dataURL. Ready for JSON.stringify().

const data = art.export()
console.log(JSON.stringify(data, null, 2))
// { exif: {...}, pxif: { pixels: [...] }, dataURL: '' }
.setPixel() setPixel(pixel?: Partial<Pixel>): void ▶

Sets a pixel at the given coordinates. Defaults: x=0, y=0, color='rgba(255,255,255,1)', size=1. If a pixel already exists at those coordinates, it's overwritten.

// Set a red pixel at (3, 5)
art.setPixel({ x: 3, y: 5, color: 'rgba(255, 0, 0, 1)' })

// Set a default white pixel at (0, 0)
art.setPixel()
.getPixel() getPixel(x?: number, y?: number): Pixel | undefined ▶

Returns the pixel at the given coordinates, or undefined if none exists. Uses O(1) Map lookup internally.

const pixel = art.getPixel(3, 5)
// { x: 3, y: 5, color: 'rgba(255, 0, 0, 1)', size: 1 }

const missing = art.getPixel(99, 99)
// undefined
.getAllPixels() getAllPixels(): Pixel[] ▶

Returns a new array of all pixel objects. Useful for iteration and rendering.

const pixels = art.getAllPixels()
pixels.forEach(px => {
  console.log(`(${px.x}, ${px.y}) = ${px.color}`)
})
Metadata Accessors get/set artist, software, imageDescription, ... ▶

All EXIF metadata fields are exposed as getter/setter pairs: artist, software, imageDescription, userComment, copyright, dateTime, and dataURL.

// Set values
art.artist = 'Pixel Master'
art.software = 'my-pixel-editor'
art.copyright = '2026 Pixel Master'
art.dateTime = new Date().toISOString()

// Read values
console.log(art.artist)           // 'Pixel Master'
console.log(art.dateTime)         // '2026-02-23T...'
